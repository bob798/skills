# Article Types

用于在 Step 0 判断文章类型，并为不同类型选择不同结构。不要把所有文章都按同一套方式写。

## 一、方法文

### 适用场景

- 用户想讲清楚一套方法、框架、判断标准。
- 目标是让读者理解并复用。

### 读者最关心什么

- 这套方法解决什么问题。
- 为什么要这样做。
- 具体怎么做。
- 哪些场景适用，哪些场景不适用。

### 推荐结构

```text
1. 先指出一个真实问题
2. 解释为什么多数人处理不好
3. 给出你的方法或框架
4. 分步骤展开
5. 给出例子或反例
6. 总结适用边界
7. 告诉读者下一步怎么做
```

### 常见错误

- 上来就讲概念，不讲问题。
- 只有框架，没有使用场景。
- 只有道理，没有步骤。
- 结尾没有行动建议。

## 二、案例复盘文

### 适用场景

- 用户经历过一个具体事件，想复盘其中的问题、判断和教训。
- 目标是让读者通过案例理解方法或认知变化。

### 读者最关心什么

- 事情是怎么发生的。
- 关键转折点是什么。
- 当时为什么会判断错。
- 这个案例能带来什么方法或教训。

### 推荐结构

```text
1. 用具体事件开头
2. 交代当时的判断
3. 说明哪里出了问题
4. 拆解真正原因
5. 提炼出可复用的方法或原则
6. 说明下次怎么避免
```

### 常见错误

- 写成流水账。
- 只讲经过，不讲判断。
- 只讲教训，不讲为什么会犯错。
- 没有从案例抽出方法。

## 三、观点文

### 适用场景

- 用户有一个明确判断，想说服读者改变看法。
- 目标是打破常见误解，建立新的理解框架。

### 读者最关心什么

- 你的观点到底是什么。
- 为什么和常见看法不同。
- 你的判断依据是什么。
- 这个观点意味着什么。

### 推荐结构

```text
1. 抛出明确观点或反常识判断
2. 说明多数人的常见理解
3. 解释为什么那个理解有问题
4. 给出你的论证
5. 用案例或事实支撑
6. 推出更大的含义
```

### 常见错误

- 观点不够明确。
- 只有态度，没有论证。
- 想讲很多观点，导致主线不清。
- 标题很强，正文很散。

## 四、指南文

### 适用场景

- 用户想让读者“照着做”。
- 目标是提供步骤、检查表、操作建议。

### 读者最关心什么

- 具体步骤是什么。
- 从哪里开始。
- 做对和做错的区别是什么。
- 常见坑有哪些。

### 推荐结构

```text
1. 先定义适用对象
2. 说明这篇能帮读者解决什么
3. 给出步骤或流程
4. 每一步解释目的和判断标准
5. 补常见错误
6. 给最小行动清单
```

### 常见错误

- 写成理念文，不像指南。
- 步骤太粗，读者不知道怎么执行。
- 缺少判断标准和避坑提示。
- 结尾没有最小行动清单。

## 五、如何判断该选哪一种

可以用下面的简化判断：

- 想讲框架和方法，用方法文。
- 想讲一次真实经历及教训，用案例复盘文。
- 想提出一个判断并说服别人，用观点文。
- 想让读者照着做，用指南文。

如果一篇文章同时具备多种属性，先判断主目标是什么，再允许局部混合。
