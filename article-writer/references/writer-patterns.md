# 写作模式参考库 · Writer Patterns

三种高质量文章写作模式的详细骨架、示例开场和关键技法。

---

## Gladwell 模式

**核心机制：认知冲突 → 机制揭示 → 更大洞察**

不从共鸣开场，从「让读者皱眉」开场。好奇心比共鸣更难放下。

### 文章骨架

```
第1段   描述一个具体场景（只描述，不解释）
第2段   抛出反直觉事实「但是，事实是……」
第3-5段  展开这个反直觉的具体案例（有画面感）
第6-8段  揭示背后机制「这是因为……」
第9-11段 机制的更大含义「这意味着……」
最后段  回扣开头场景，用新视角重看它
```

### 示例开场

> 2022年秋天，一个普通的下午，我把一个产品需求发给了 Claude。 我以为我说清楚了。 两小时后，我看到了一个完全符合描述、但完全不是我想要的东西。
> 
> 那一刻我以为是 AI 的问题。 后来我才知道，问题从来不在 AI。

### 关键技法

- **具体性**：不说「某个项目」，说「2022年秋天」「上海办公室」
- **延迟解释**：把「为什么」延迟到读者已经困惑时才给出
- **对称回扣**：结尾必须回到开头场景，用新眼光重看它
- **一个贯穿故事**：全文靠一个具体故事驱动，不是多个例子堆砌

### 适用场景

强反直觉观点 · 「大多数人理解反了」型内容 · 有一个核心案例可以展开

## Paul Graham 模式

**核心机制：思维过程可见 → 读者跟着一起想 → 结论是「发现」不是「接受」**

PG 不装作一开始就知道答案。他写的是「我是怎么想到这个的」。

### 文章骨架

```
第1段   「我一直以为 X」（旧认知，读者可能共享）
第2段   「直到 Y 发生了」（触发事件，必须具体）
第3-4段  「我开始重新想这件事」（思维过程，不假装跳跃）
第5-6段  「我注意到一个模式」（洞察浮现）
第7-8段  「这让我想到 Z」（延伸推导）
最后段  「但还有一个问题我没想清楚……」（开放结尾）
```

### 示例开场

> 我一直以为管理 AI 团队，和管理人类团队没什么本质区别——给任务，定目标，验收结果。
> 
> 我错了三个月。
> 
> 不是错在方向上，而是错在一件更基础的事：我以为我在管理一个「能理解我」的团队，但其实我在管理一个「只执行我说的话」的系统。这两件事，差距比我想象的大得多。

### 关键技法

- **一人称思维**：「我以为」「我发现」「我开始想」，保持第一视角全程
- **主动暴露错误**：承认旧认知是错的，这建立信任而不是削弱权威
- **不假装跳跃**：每一步推导都写出来，不突然跳到结论
- **开放结尾**：不强行总结，留一个让读者思考的问题

### 适用场景

探索性观点 · 个人反思 · 「我改变想法了」型内容 · 适合建立个人 IP 的文章

## Ben Thompson 模式

**核心机制：建立框架 → 用框架推导 → 做出可验证预测**

读者跟着学会用框架看问题，而不只是接受结论。系列文章的标准结构。

### 文章骨架

```
第1段   当前无争议的事实（不抬杠的起点）
第2-3段  现有解释的不足（为什么需要新框架）
第4-5段  建立新框架「我认为真正的原因是……」
第6-7段  用框架解释现象「用这个框架看，就能理解为什么……」
第8-9段  用框架做预测「所以接下来会发生……」
最后段  与读者的约定「下一篇我们看这个预测是否成立」
```

### 示例开场

> 过去三年，几乎每家公司都宣布了自己的「AI 战略」。大多数战略的核心是同一件事：用 AI 替代人工。
> 
> 我认为这个框架是错的。不是方向错，是层级错了。
> 
> 真正的变化，不发生在执行层，而发生在协作语言层。

### 关键技法

- **框架先行**：文章核心是框架，结论是框架推出来的
- **明确预测**：用「会发生 X」不用「可能发生 X」，让预测可验证
- **系列连接**：每篇结尾明确说下一篇要检验什么，制造连载感
- **框架命名**：给框架一个好名字（「协作语言论」「聚合理论」），方便读者传播

### 适用场景

行业分析 · 趋势判断 · 系列文章 · 「我有一个模型可以解释这一切」型内容

## 混合使用指南

三种模式可在同一篇文章中混合：

```
开场  → Gladwell（反直觉钩子，3段内让读者皱眉）
中段  → PG（思维过程可见，建立信任）
结尾  → Thompson（框架+预测，留住读者看下一篇）
```

**判断用哪种开场：**

- 有一个强反直觉案例 → Gladwell
- 是自己的反思和改变 → PG
- 要建立分析框架 → Thompson
- 不确定 → PG 最安全，最容易写，最建立个人信任

## 反面教材：常见错误开场

```
❌「今天我想和大家聊聊……」
   → 没有钩子，读者立刻滑走

❌「随着 AI 的发展，越来越多的人开始……」
   → 空洞的背景铺垫，一句废话

❌「本文将从三个方面分析……」
   → 报告体，不是文章体

✅ 直接进入具体场景或反直觉事实
```
